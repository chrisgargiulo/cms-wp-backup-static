The Essentials To Planning a Design Project<\/h3>\n\n\n\n

Conceptual plans are the blueprints for any design project. Conducted early in the design process, they can include any number of documents and exercises that will be helpful in communicating ideas to all stakeholders and instrumental in insuring a quality end product and user experience. The primary goal for is to map out the project using a user-centered (target audience) approach. Planning documents should try to be style\/design independent, with a focus on the user\u2019s (target audience) experience, goals, & tasks to help determine the best possible presentation structure and content hierarchy.<\/p>\n\n\n\n\n\n\n\n

Conceptual Deliverables<\/h2>\n\n\n\n

Conceptual plans may vary from project-to-project and medium to medium. Below are a few of the most commonly used deliverables for web, print, and motion graphic design projects.<\/p>\n\n\n\n

Web Projects<\/h2>\n\n\n\n

Key Deliverables:<\/em><\/p>\n\n\n\n

  • Site Map(s)<\/li>
  • Lo-Fi UX Wireframes (e.g. Adobe XD Prototypes)<\/li>
  • Style Tiles\n
    • Color Studies<\/li>
    • Type Studies<\/li><\/ul>\n<\/li><\/ul>\n\n\n\n

      Print Projects<\/h2>\n\n\n\n

      Key Deliverables:<\/em><\/p>\n\n\n\n

      • Sketches<\/li>
      • Lo-Fi Wireframes\/Diagrams<\/li>
      • Style Tiles\n
        • Color Studies<\/li>
        • Type Studies<\/li><\/ul>\n<\/li><\/ul>\n\n\n\n

          Motion Projects<\/h2>\n\n\n\n

          Key Deliverables:<\/em><\/p>\n\n\n\n

          • Writing\/Planning related:\n
            • Synopsis<\/li>
            • Outline<\/li><\/ul>\n<\/li>
            • Pre-Visualization related:\n
              • Storyboards<\/li>
              • Story Reel<\/li><\/ul>\n<\/li><\/ul>\n\n\n\n

                Other Considerations<\/h3>\n\n\n\n

                Related Deliverables:<\/em><\/p>\n\n\n\n

                • Content Strategies\n
                  • User flow charts<\/li>
                  • Content language (nomenclature) options and writing styles (high risk? low risk? informal? formal?)<\/li>
                  • Content docs, maps, and matrices<\/li><\/ul>\n<\/li>
                  • Visual Studies:\n
                    • Mood Boards<\/li>
                    • List of Potential UI Components and Design Patterns<\/li>
                    • Visual and Technical Tests (as many as needed to help define the visual style)<\/li><\/ul>\n<\/li><\/ul>\n\n\n\n
