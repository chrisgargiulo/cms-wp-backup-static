Reusable Solutions to Known Problems<\/h3>\n\n\n\n

In web design, a design pattern is not a visual<\/em> “pattern” (e.g. a background image of repeating shapes and colors). It’s easy to see how one might immediately think of this when they put the words design<\/em> and pattern<\/em> together. Originating from software engineering, a web (UX, UI) design pattern<\/strong> is a reusable solution<\/em> to a known problem<\/em>. <\/p>\n\n\n\n

User interface design patterns are descriptions of best practices within user interface design. They are general, reusable solutions to commonly occurring problems. As such, they form the backbone of \u201ctechnical support.\u201d However, as design patterns can be applied to a wide variety of instances, designers should adapt them to the specific context of use within each design project.<\/p>Interaction Design Foundation<\/a><\/cite><\/blockquote>\n\n\n\n

            Design Pattern Example: Site Headers<\/h3>\n\n\n\n

            Site Headers & Site Branding, Identity, & Navigation<\/h4>\n\n\n\n

            There are many different ways to structure and style your site header. Since it is is at the top of the interface, the site header plays an important role in establishing the site brand, identity, and navigational system. Below are a few different options to consider when customizing your site header.<\/p>\n\n\n\n

            <\/div>\n\n\n\n

            Site Headers & Information Architecture (IA)<\/h4>\n\n\n\n

            When it comes to site headers, one of the first decisions that web designers need to make is what content will be included within the header. Common elements and related concerns often times include:<\/p>\n\n\n\n

            • Site Name
              • often times located in the top-left corner<\/li>
              • clickable\/linkable to the home page (e.g. href=”index.html”)<\/li><\/ul><\/li>
              • Logo
                • if there are multiple versions, which version of the logo should be included here?<\/li>
                • Logo + Site name? Logo only?<\/li>
                • Responsive logo? (e.g. see responsive logos<\/a>)<\/li><\/ul><\/li>
                • Navigation (Main Menu)
                  • How many sub pages should be included?<\/li>
                  • Should a link to “Home” also be included within the nav?<\/li>
                  • Should third-level pages also be included (e.g. via drop down menus)?<\/li>
                  • Any call-to-action buttons (e.g. “Sign in”)<\/li><\/ul><\/li><\/ul>\n\n\n\n
                    \n\n\n\n
                    <\/div>\n\n\n\n

                    Site Headers & User Experience Design (UX)<\/h4>\n\n\n\n

                    How will the user interact with the header? How should the site header behave? Should it always be fixed at the top? or scroll away? Should it appear large at first when the user is at the top of the page, but then shrink in size as the user scrolls down? Should the header expand when the user hovers or clicks on certain parts of it (e.g. navigation items)? Where should the site identity\/logo be placed? Top center? Top Left? How will users know what page they are on? Should the site header include a visual indicator to provide feedback to the user what page you are on? Will there be any changes to the user experience for small screens? extra large screens?<\/p>\n\n\n\n

                    \n\n\n\n
                    <\/div>\n\n\n\n

                    Site Headers & User Interface Design (UI)<\/h4>\n\n\n\n

                    What color should the site header be? What color should the text be? What typography works well to both communicate the brand and be usable\/readable\/legible? Should the site identity text be different (e.g. larger? bolder?) than the navigation text? What should the layout be? Header left-aligned and navigation right-aligned? Both centered and stacked? Should each element align to a grid? Should they align to a baseline? How should the site look on small screens? extra large screens?<\/p>\n\n\n\n
