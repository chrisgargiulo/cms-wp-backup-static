One of the best ways to learn about content management systems (CMS) is to set one up from scratch. In this lesson, we are going to use WordPress<\/a> to set up a small, simple CMS-driven “test” web site for learning purposes.<\/p>\n\n\n\n

A Content Management System<\/strong> (CMS) is a computer application that supports the creation and modification of digital content. It is typically used to support multiple users working in a collaborative environment. Most CMSs include Web-based publishing, format management, history editing and version control, indexing, search, and retrieval. By their nature, content management systems support the separation of content and presentation.
\u2013 Wikipedia<\/a><\/p><\/blockquote>\n\n\n\n

WordPress<\/strong> is a free and open-source content management system (CMS) based on PHP and MySQL.
\u2013 Wikipedia<\/a><\/p><\/blockquote>\n\n\n\n

                                          Customize Your WordPress Site<\/h2>\n\n\n\n

                                          There are many different ways to customize a WP site, below are are few known techniques that you should be familiar with as you consider your best options per project:<\/p>\n\n\n\n

                                          • You can customize a theme via Customizer
                                            • Appearance > Customize
                                              • Browse the built-in theme settings and experiment with the theme-supported settings<\/li><\/ul><\/li><\/ul><\/li>
                                              • You can change the Theme
                                                • Appearance > Themes > Add New
                                                  • Search or browse for a new third-party theme <\/li><\/ul><\/li><\/ul><\/li>
                                                  • You can modify a Theme
                                                    • Manually edit a theme’s css and php files directly via Appearance > Editor (not recommended*)<\/em><\/li>
                                                    • Manually edit a theme’s css and php files locally then update\/upload via FTP (not recommended*)<\/em><\/li><\/ul><\/li>
                                                    • You can create a Child Theme
                                                      • Create a new theme directory with only the files that you want to override (highly recommended)<\/em><\/li><\/ul><\/li>
                                                      • You can create your own Custom Theme
                                                        • This option provides the most control, but it also the most costly in terms of development time (advanced, but highly recommended)<\/em><\/li><\/ul><\/li><\/ul>\n\n\n\n

                                                          * It is never recommended to edit a found parent theme’s files because all changes\/modifications will be lost if\/when the theme is updated. A safer approach is to use a child theme or to create a custom parent theme.<\/em><\/p>\n\n\n\n
