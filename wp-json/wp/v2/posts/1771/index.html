Pre-launch QA Diligence<\/h3>\n\n\n\n

Prior to the launch of web sites both big and small, there are several techniques and best practices to help ensure the delivery of a quality product that is performant and bug free. Often times addressed toward the end of a project, one of the most commonly used terms to describe this step of the process is quality assurance<\/em>, or QA<\/em>.<\/p>\n\n\n\n\n\n\n\n

Quality assurance (QA)<\/strong> is a way of preventing mistakes and defects in manufactured products and avoiding problems when delivering solutions or services to customers
\u2013 Wikipedia<\/a><\/p><\/blockquote>\n\n\n\n

Bug Tracking<\/h2>\n\n\n\n

Validate, Fix Errors, Manage Enhancements<\/em><\/p>\n\n\n\n

  • Track and Prioritize your Bugs and Tasks
    • Online tools: Google Spreadsheets (Example QA Bug Tracking Sheet<\/a>), Trello Boards (Example Trello Bug Tracking Board<\/a>), Github Issues<\/a>, etc. <\/li><\/ul><\/li>
    • Validate your code
      • Validate HTML<\/a><\/li>
      • Validate CSS<\/a><\/li>
      • Validate Accessibility<\/a> (Section 508 compliance)
        • Provide “equivalent” experience to all users. For example:
          • make sure all img alt tags are appropriate<\/li>
          • make sure the site can be viewed\/navigated via keyboard (hitting the tab key)<\/li><\/ul><\/li>
          • Check out online resources, such as: The W3C Web Accessibility Initiative (WAI)<\/a> and their list of resources<\/a>, the A11Y Project<\/a>, ARIA<\/a>, etc. <\/li><\/ul><\/li><\/ul><\/li>
          • Test in multiple Platforms, Browsers, Screen Sizes, and Devices
            • Example of online Tools: Browserling<\/a>, Netrenderer<\/a>, Sauce Labs<\/a>, Browserstack<\/a>, Browsershots<\/a>, Chrome’s Device Mode<\/a> <\/li><\/ul><\/li><\/ul>\n\n\n\n

              Performance Enhancements<\/h2>\n\n\n\n

              Techniques to improve the speed of your site<\/em><\/p>\n\n\n\n

              • Test your site at web.dev <\/a>and Google Page Speed Insights<\/a><\/li>
              • Conduct a site audit for K size of all assets<\/li>
              • Optimize all media (e.g. images, videos, etc). Example: manually adjust via Photoshop’s Save for the web (legacy).
                • Recommended Image Formats:
                  • jpg (for images\/raster photos with lots of color)<\/li>
                  • gif (for images\/illustration with under 256 colors)<\/li>
                  • png-24 (for images that require alpha transparency <\/li><\/ul><\/li>
                  • Consider using a CDN that provides responsive media out-of-the-box (e.g. Cloudinary<\/a>)<\/li><\/ul><\/li><\/ul>\n\n\n\n
                    • Minify all local dependencies (html, css, and js)<\/li>
                    • Considerations:
                      • Consider using a CDN (e.g. Netlify<\/a> for static site hosting)<\/li>
                      • Consider using a critical css<\/a> file<\/li>
                      • Consider file compression (e.g. gzip) <\/li><\/ul><\/li><\/ul>\n\n\n\n

                        Due Diligence<\/h2>\n\n\n\n

                        Style guides, code clean-up, and more…<\/em><\/p>\n\n\n\n

                        • Refactor all code by removing any redundant or unnecessary code.\n
                          • Remove unused markup (html)<\/li>
                          • Remove unused styling (css)<\/li>
                          • Concatenate your Javascript into one js file<\/li><\/ul>\n<\/li>
                          • Consider creating in-house style guidelines & documention (for internal developer use and\/or client use).<\/li><\/ul>\n\n\n\n

                            Quick Checklist<\/h2>\n\n\n\n
                            1. Is your HTML & CSS validating with no errors?<\/li>
                            2. Is your site Section 508 Compliant?<\/li>
                            3. Have you tested on Multiple Platforms, Browsers, Devices, and Screen Sizes?\n
                              • Mac and PC<\/li>
                              • Chrome, Firefox, Safari, IE, Edge<\/li>
                              • Large Monitor, Laptop Display, Tablet, Mobile (including portrait & landscape views)<\/li>
                              • iOS and Android<\/li><\/ul>\n<\/li>
                              • Have you done a site audit and optimized all assets and media (images, videos, etc.)?<\/li><\/ol>\n","protected":false},"excerpt":{"rendered":"
